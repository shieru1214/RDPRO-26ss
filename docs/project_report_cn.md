## 一、一句话讲清 Jiaozi 是什么

**Jiaozi 是一个"AI 视觉模型的选型顾问 + 自动出代码工具"。**

打个比方：你想装修房子（= 训练一个能识别图片的 AI 模型），但你不懂装修。
Jiaozi 做的事是：

1. **听懂你的需求**（"我要识别叶子病害，准一点，但要能跑在手机上"）
2. **看一眼你的房子**（你的图片数据集——多少张、多少类别、清晰度如何）
3. **从一个装修知识库里挑出最合适的整套方案**（用哪个 AI 模型骨架、配什么部件、怎么训练）
4. **直接把施工图画好**（生成一整套可以直接运行的训练代码）

用户只要给一句话需求 + 一个数据集，Jiaozi 就能输出"该用什么模型 + 为什么 + 可运行的代码"。

> 名词解释：
> - **模型/AI 模型**：能识别图片的程序。需要用数据"训练"出来。
> - **训练**：拿一堆带标注的图片喂给模型，让它学会识别。
> - **数据集**：你的那堆带标注图片。

---

## 二、为什么需要它（解决什么痛点）

普通人/中小团队想做一个图像识别 AI，会卡在三个地方：

| 痛点 | 具体表现 |
|---|---|
| **不知道选哪个模型** | AI 模型有几十种，各有擅长，选错了又慢又不准 |
| **不知道怎么配参数** | 同一个模型，参数配错（比如"学习速度"设错）结果天差地别 |
| **不会写训练代码** | 即便选对了，把它跑起来还要写几百行代码 |

Jiaozi 把这三件事**自动化**了：选型 + 配参 + 出代码，一条龙。

## 三、整体怎么工作（四步流水线）

Jiaozi 内部分成四个模块，像流水线一样接力。我们用"装修"类比：

```
你的一句话需求 ─┐
              ├─→ ① 听懂需求 ─┐
你的数据集   ─┘            ├─→ ③ 选型大脑 ─→ ④ 出施工图（代码）
              ② 看数据 ───┘
```

| 模块 | 通俗说 | 干的事 |
|---|---|---|
| **模块 1：听懂需求** | 把人话翻译成清单 | 从"要准、要省、能上手机"提取出：任务类型、看重准还是快、有没有特殊约束 |
| **模块 2：看数据** | 给数据集做体检 | 数了多少张图、多少类别、清晰度、是否"偏科"（某类特别多） |
| **模块 3：选型大脑** | 从知识库挑方案 | 综合需求 + 数据，选出最合适的模型 + 部件 |
| **模块 4：出施工图** | 把方案变成代码 | 生成可直接运行的训练/评估/预测代码 |

下面逐个讲，并解释**每个设计选择的原因**。

## 四、逐模块详解 + 为什么这么设计

### 模块 1：听懂需求

**做什么**：用户输入一句自然语言（"识别木薯叶病害，准确率优先，类别不均衡"），
模块 1 借助一个**大语言模型**（就是 ChatGPT 那类会读会写的 AI）把它翻译成一张结构化清单：

- 任务类型（分类 / 检测 / 分割 / 特征提取）
- 优先级（看重速度 / 准确率 / 平衡）
- 约束（要不要实时、要不要上边缘设备、类别均不均衡……）
- **评估指标**（按什么打分：准确率 / AUC / F1 / 分级一致性……）

**为什么这么设计**：
- **为什么用大语言模型**：用户的需求是自由的人话，规则匹配处理不了"含糊表达"。大语言模型最擅长把人话变结构化。
- **为什么最近加了"评估指标"识别**：以前系统不管用户怎么说，都默认按"准确率"打分。但有些任务（比如医学按 AUC、病情分级按"分级一致性"）用准确率是不对的。现在用户说"按 AUC 评"，系统就真的用 AUC——**否则选出来的方案是针对错误目标优化的**。

### 模块 2：给数据集做体检

**做什么**：加载用户的数据集，统计：总图数、类别数、每类多少张（判断是否偏科）、
图片清晰度（宽高）、是彩色还是灰度。

**为什么这么设计**：
- **为什么要看数据而不是只听需求**：用户常常不知道自己数据的关键事实。比如"我数据很多"——
  但 2 万张图分给 200 个类别，每类才 100 张，其实是**小数据**。数据规模直接决定该用大模型还是小模型、要不要防过拟合。
- **一个我们修过的效率问题**：体检时原本会把每张图**重复解码 3~4 遍**（为了分别数尺寸、颜色、格式），
  9000 张图就解码 3 万多次，要好几分钟。我们改成**只扫一遍 + 抽样**，从几分钟降到几秒——
  **这不影响结果，纯粹是把笨办法改聪明。**

> 名词解释：
> - **过拟合**：模型把训练图"背下来"了，但换新图就不认识——数据太少时最容易发生。
> - **大模型 vs 小模型**：大模型更准但更慢更贵、小数据上更容易过拟合；小模型相反。

### 模块 3：选型大脑（核心）

**做什么**：综合模块 1+2 的信息，从一个**知识库**里选出最合适的"整套方案"：
模型骨架 + 预训练权重 + 输出部件 + 损失函数 + 优化器 + 训练策略。

**这个知识库长什么样**：我们人工整理了 14 个主流模型骨架、22 个现成的"预训练权重"
（别人在海量数据上训练好的、可以直接拿来微调的"半成品"）、以及配套部件。
它们之间的关系（谁能配谁、谁是谁的替代）也都记录在一张"关系图"里。

**为什么这么设计**：
- **为什么用人工知识库而不是让 AI 现场上网搜**：知识库**可控、可解释、稳定**——
  我们能说清"为什么选这个"，而上网现搜是黑盒、不稳定。（这点是我们和竞品的关键差异，见第五节。）
- **怎么选的（多个信号综合打分）**：按数据规模匹配、按"看重准还是快"匹配、按约束过滤，
  再加一个"语义相似度"（把用户描述和模型介绍做匹配）。最后综合排序取前 3。

**我们给它加的三个重要能力**（都有明确原因）：

1. **约束感知 / 预算意识** ✅ 已上线
   - **问题**：以前系统不管你说"要上手机"，都可能推一个几亿参数的大模型，根本塞不进手机。
   - **做法**：给每个模型标注了"体量"（参数量、计算量），用户可以给一个**预算**
     （比如"不超过 1200 万参数"）。系统会**自动把超预算的砍掉，并自动降级到能塞进预算的小版本**
     （比如 ResNet50 太大 → 自动换成 ResNet18）。
   - **为什么重要**：真实部署永远有约束（手机内存、延迟）。一个"在约束内最好"的方案，
     比"裸分最高但根本部署不了"的方案有用得多。

2. **会积累、会解释的推荐器（"自学习 RAG"）** 🔬 在做 / 规划中（实验分支）
   - **问题**：纯靠人工规则选型不够聪明，而且**不会从经验中学习**——同样的错会犯一百遍。
   - **做法**：每跑完一次训练，系统把"这个数据集长这样 + 用了这个方案 + 得了多少分 + 花了多少成本"
     记进一个**记忆库**。下次遇到相似的数据集，就**优先推荐过去真的好用的方案**，
     并且**给出理由**（"推荐它，因为在相似的 beans 数据集上它得了 0.86 分"）。
   - **冷启动怎么办**（一开始记忆是空的）：我们用一个叫 **LogME** 的便宜方法——
     不用真训练，就能在你的数据上快速估算"哪个模型骨架最匹配"，用它当起步信号。
   - **为什么重要**：这是 Jiaozi **越用越聪明**的来源，也是和竞品最大的差异（竞品每次从零开始、还健忘）。

3. **会推荐"超参数"的配方层** 🔬 在做 / 规划中（实验分支）
   - **名词**：超参数 = 训练时的设置（学习速度、训练轮数、图片分辨率等），配错了结果会很差。
   - **问题**：以前这些设置全是**写死的固定值**，不管什么模型什么数据都一样——很不聪明。
   - **做法**：做了一个"配方"，按（模型类型、训练策略、数据规模、图片清晰度）给出合适的设置。
     比如"微调大型 Transformer 模型 → 骨架部分用很慢的学习速度，否则会把预训练学到的东西忘光"。
   - **为什么这些规则靠谱**：超参数的很多规律是**机制性的、业界公认的**（不像"选哪个模型"那么靠猜），
     所以规则化是可靠的。
   - **未来**：接口已经预留，以后可以让大语言模型来提议这些设置（它读过海量训练配方），
     但会被规则"护栏"约束住，不会失控。

### 模块 4：出施工图（生成代码）

**做什么**：把模块 3 选好的方案，变成一整套**可以直接运行**的代码：
训练、评估、预测、运行脚本、配置文件等。

**为什么这么设计**：
- **大部分代码用"模板"生成**：训练/评估这套骨架是固定可靠的，用模板保证**不会出错**。
- **只有"模型结构"那一小块可以让大语言模型写**（写不好就退回模板）。
  - **为什么这样分**：把可靠性留给模板，把灵活性留给 AI。
- **我们最近的两个改进**：
  - **出错先纠错再放弃**：以前 AI 写的代码一报错就直接退回模板。现在会**把错误告诉 AI、让它改，
    改几次还不行才退回**——更聪明，但**有次数上限**（控制成本），不是无限折腾。
  - **兼容新模型**：某些新版 AI（如 gpt-5.x）对调用参数有特殊要求，我们做了自动适配，不再报错。

## 五、我们的核心差异 / 凭什么有价值

市面上有一个很强的竞品叫 **MLE-STAR**（谷歌出的）。诚实地说，**论"裸分"（纯比谁训练出的模型更准），
它比我们强**——因为它会**反复试错、不断搜索、还会把多个模型组合**（这叫 ensemble，集成）。

但它的工作方式决定了三个**结构性弱点**，而这正是 Jiaozi 的位置：

| 维度 | MLE-STAR（竞品） | Jiaozi |
|---|---|---|
| **成本** | 每个任务反复调用 AI + 反复训练，**很贵很慢** | 基本一次搞定，**便宜几十倍** |
| **可解释** | 黑盒，给你结果但说不清为什么 | **每个推荐都有理由**（数据信号 + 历史证据） |
| **会不会积累** | 每个任务从零开始，**做完就忘** | **越用越准**（记忆库），边际成本趋近 0 |
| **约束感知** | 只追求高分，不管能不能部署 | **在预算内选最优**，天生适配部署 |

> **成熟度说明**：上表中 **约束感知 ✅ 已上线**；**可解释、会积累** 来自"自学习推荐器"，
> 目前 **🔬 在实验分支、属下一阶段重点**。也就是说这张对比表里，"低成本 + 守预算"现在就成立，
> "可解释 + 越用越准"是已有原型、正在推进的方向——这是我们的**路线图差异**，不是已完全落地的现状。

**我们的价值主张（一句话）**：
> 不和它拼"谁的分最高"——那要烧很多钱。我们占的是它结构上够不到的角落：
> **用极少的成本（少量 AI 调用、不反复试错），给出接近的、可解释的、而且越用越准的方案。**

打个比方：MLE-STAR 像一个会"暴力试一百遍"的天才工程师，结果好但又贵又慢又不解释、还记不住经验。
Jiaozi 像一个**懂行、便宜、能讲清道理、而且越用越懂你**的顾问。对绝大多数"不想烧钱反复调参、
只想要一个能打的方案"的用户，后者更实用。

> **重要的诚实**：我们也评估过"模型汤""集成"这类提分技巧，结论是它们**是公开技术、竞品也能用，
> 而且和我们的成本优势冲突**，所以不把它们当卖点——这种"试过并判断不划算"的分析本身也是项目的严谨之处。

## 七、给非技术同学的总结

1. **它解决什么**：让不懂 AI 的人，也能"一句话 + 一个数据集"就得到一套能用的图像识别方案 + 代码。
2. **怎么做到**：四步流水线——听懂需求 → 看数据 → 选型大脑 → 出代码。
3. **凭什么有价值**：不拼"最贵最准"，而是**便宜、可解释、守预算、越用越准**——
   这是大竞品结构上做不到的角落。
4. **现在到哪了**：核心链路全通，能在云端跑；下一步是用真实数据验证"越用越准"，并做出"质量 vs 成本"对比图。

> 一句话电梯陈述：
> **"Jiaozi 是图像 AI 的选型顾问——一句话需求就给你可运行的方案，便宜、能解释、还越用越懂你；
> 我们不和烧钱的暴力竞品拼裸分，而是占据它够不到的'高性价比 + 可解释 + 会积累'的角落。"**
